English GitHub Twitter

将 Figma / iOS 的 corner radius + smoothing 转换为 CSS border-radius + corner-shape: superellipse(K)。

什么是 Smooth Corner

标准 border-radius 绘制的是圆弧，圆弧与直边相接的位置会出现突然的曲率变化。Figma 的 Corner Smoothing 滑块和 iOS 的 UIBezierPath .continuous 风格会使用超椭圆曲线替代圆弧，让曲率过渡更连续。smoothing 参数控制这种过渡的范围：0 表示纯圆弧，1 表示最大平滑。

新的 CSS 属性 corner-shape: superellipse(K) 可以原生表达这类效果。本库负责把设计工具里的 radius + smoothing 值转换成浏览器需要的补偿后 border-radius 和 corner-shape: superellipse(K)。

安装

npm i @qiuyedx/smooth-corners

纯 ESM、零依赖、无需构建步骤。提供三层 API：

  • CSS 变量 API - 适合大多数场景。只需要 radius 和 smoothing，能直接用于 React/Vue 组件。
  • 尺寸感知计算 - 当圆角半径可能接近元素短边一半时使用，会自动约束以避免溢出。
  • Observer - 当你希望自动跟踪元素尺寸时使用，无需手动计算。

CSS 变量 API

smoothCorners(radius, smoothing?) 会返回一组 CSS 自定义属性，配合 .smooth-corners 类使用。该类通过 @supports 做渐进增强：支持 corner-shape 时使用补偿后的半径 + 超椭圆；不支持时回退到原始半径。

把这些规则加入全局样式表即可，也可以直接使用导出的 smoothCornersCSS：

.smooth-corners {
  border-radius: var(--sc-r);
}
@supports (corner-shape: superellipse(2)) {
  .smooth-corners {
    border-radius: var(--sc-i);
    corner-shape: var(--sc-s);
  }
}

React

import { smoothCorners } from '@qiuyedx/smooth-corners';

<div className="smooth-corners" style={smoothCorners(30)} />

Vue

import { smoothCorners } from '@qiuyedx/smooth-corners';

<div class="smooth-corners" :style="smoothCorners(30)" />

Vanilla JS

import { applySmooth, smoothCorners } from '@qiuyedx/smooth-corners';

applySmooth(element, smoothCorners(30));

返回值

  • --sc-r - 原始半径，用于回退
  • --sc-i - 补偿后的半径，用于超椭圆模式
  • --sc-s - superellipse(K) 值

smoothing 默认值为 0.6，对应 Figma 的 60% Corner Smoothing。设置为 0 表示纯圆弧，设置为 1 表示最大平滑。

尺寸感知计算

当你知道元素的实际尺寸时，可以使用 computeSmoothCorners(width, height, radius, smoothing) 获得更准确的结果。它会把半径约束到 min(width, height) / 2 以内，并在空间不足时逐步把 smoothing 压缩到 0，避免相邻圆角互相重叠。 

import { computeSmoothCorners } from '@qiuyedx/smooth-corners';

const result = computeSmoothCorners(180, 120, 30, 0.6);
// {
//   radius: 30,             // 目标圆角半径
//   compensatedRadius: ..., // 平滑算法计算出的补偿半径
//   k: ...,                 // 超椭圆指数
//   smoothing: ...,         // 实际生效的 smoothing，空间不足时会自动降低
// }

Observer

Observer 内部使用 computeSmoothCorners，会自动读取元素尺寸并应用 CSS 自定义属性。它同时支持声明式 HTML 属性和编程式调用，并在元素尺寸变化时自动重新计算。基础样式 .smooth-corners 会在首次使用时自动注入，不需要手动设置。

声明式

给元素添加 data-smooth-corners 属性，然后导入 smooth-corners/declarative 即可启用自动扫描。元素新增、移除和属性变化都会被 MutationObserver 捕获。 

<div data-smooth-corners="30">...</div>

<script type="module">
  import '@qiuyedx/smooth-corners/declarative';
</script>

编程式

使用 observe(el, opts) / unobserve(el) 直接控制单个元素。如需手动管理生命周期，可从 smooth-corners/observer 导入 startAutoObserve() / stopAutoObserve()。 

import { observe, unobserve } from '@qiuyedx/smooth-corners/observer';

observe(el, { radius: 30, smoothing: 0.6 });
unobserve(el);

算法

给定原始半径 r 和平滑值 s，计算 corner-shape: superellipse(K) 所需的 K。

1. 补偿半径

平滑会把曲线向外延展，因此需要更大的 border-radius 进行补偿。

compensated = r * (1 + s)

尺寸感知版本会把延展量限制在可用空间内：

extension = min(r * s, limit - r)
compensated = r + extension

2. 超椭圆指数 n

超椭圆方程 |x|^n + |y|^n = 1 在 n = 2 时表示圆。这里使用闭式公式求解 n：

d = 1 - 1/sqrt(2) ~= 0.293
u = 1 - (r * d) / compensated
n = ln2 / ln(1/u)

3. CSS K 值

把 n 映射到 CSS corner-shape 参数：

K = log2(max(2, n))

浏览器支持

corner-shape: superellipse(...) 是新的 CSS 属性，Chrome 138+ 支持。本库所有 API 都使用渐进增强：

  • 支持时 - 使用补偿半径 + superellipse(K) 获得真正的平滑圆角
  • 不支持时 - 回退到标准 border-radius 和原始半径，仍然保留圆角，但没有平滑过渡

所有 API 都通过 CSS @supports 自动切换，无需 polyfill 或条件逻辑。
